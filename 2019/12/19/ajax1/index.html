AJAX学习第一天

Calcc · 2019-12-19 · 次阅读

今日目标

  1. 了解AJAX 的使用场景 和 目的
  2. 了解 AJAX 的原理
  3. 学会使用 XHR 对象发送 get 请求
  4. 学会学会使用 XHR 对象发送 post 请求
  5. 知道常见的 ajax 错误状态码
  6. 封装 实现 Ajax 函数 （较难）

Ajax的介绍

  1. 使用场景

    1. 搜索建议的功能

    2. 注册时校验用户名是否重复

    3. 页面的滚动加载数据

  2. 本质：在向服务端请求数据

  3. 目的：提升用户使用网站的体验， 等待时间更少，体验更好

使用Ajax 的步骤

  1. 创建异步请求对象
  2. 调用 open 方法
  3. 调用send 发送这个请求
  4. 接收响应的内容
    // 1.创建ajax对象
        var xhr = new XMLHttpRequest();
        // 2.告诉Ajax对象要向哪发送请求，以什么方式发送请求
        // 1)请求方式 2)请求地址
        xhr.open('get', 'http://localhost:50003/first');
        // 3.发送请求
        xhr.send();
        // 4.获取服务器端响应到客户端的数据
        xhr.onload = function (){
            console.log(xhr.responseText)
        }

注意：服务器端需要配置相应的路由文件

    // 引入express框架
        const express = require('express');
        // 创建web服务器
        const app = express();
            app.get('/first',(req,res)=>{
                res.send('hellow,word')
            })
            // 监听端口
        app.listen(50003);
        // 控制台提示输出
        console.log('服务器启动成功');

发请求的四个要素

  1. 请求方式 get post
  2. 请求的地址 URL
  3. 请求体 requestBody
  4. 请求头 requestHeader

post 请求方式一

  1. 请求头：content-type: application/x-www-form-urlencoded
  2. 请求体： a=b&b=c 使用这样的字符串
 // 收集用户输入的参数 
var userInput = document.getElementById('userName');

var ageInput = document.getElementById('age');

// 拼接参数 
var params = `username=${userInput.value}&age=${ageInput.value}`

var xhr = new XMLHttpRequest();

// 请求的方式和地址
xhr.open('post', 'http://localhost:3000/post');

// 设置请求头
xhr.setRequestHeader('Content-Type', 'application/x-www-form-urlencoded');

// 设置请求体
xhr.send(params);

xhr.onload = function() {
    console.log();
    var obj = JSON.parse(xhr.responseText);

    console.log(obj)
}

post 请求方式二

  1. 请求头: application/json
  2. 请求体：JSON.stringify()
// 收集用户输入的参数 
var userInput = document.getElementById('userName');

var ageInput = document.getElementById('age');

// 拼接参数 
var params = JSON.stringify({
    username:userInput.value,
    age:ageInput.value
})

var xhr = new XMLHttpRequest();

// 请求的方式和地址
xhr.open('post', 'http://localhost:3000/postjson');

// 设置请求头
xhr.setRequestHeader('Content-Type', 'application/json');

// 设置请求体
xhr.send(params);


xhr.onload = function() {
    var obj = JSON.parse(xhr.responseText);
    console.log(obj)
}

旧的事件

  1. onreadystatechange 在请求状态发生变化时，触发
  2. readyState 的几个值
值 状态 说明
0 UNSENT 代理被创建，但尚未调用 open() 方法。
1 OPENED open() 方法已经被调用。
2 HEADERS_RECEIVED send() 方法已经被调用，并且头部和状态已经可获得。
3 LOADING 正在接收数据
4 DONE 请求结束 ，数据接收完成

错误处理的技术

  1. 网络连通的时候
    1. 400 提交的数据 有问题
    2. 404 请求地址写错了，自已的问题
    3. 500 后端的代码写错了，后端开发的问题
  2. 网络中断
    1. 网线断了，路由器关机
    2. 服务器关了

同步与异步

  1. 异步 代码，一般不关注函数的返回值，使用一个事件处理函数得到结果
  2. 同步 代码，一般会接收函数的返回值

Ajax 封装

具体步骤：

  1. 基本功能：先定义一个 ajax 函数 ，传递一个对象进入，包含了 url, type, success 属性，发送一个 Get 请求，响应后调用 success 函数，并传递 响应结果

  2. 传递参数：在参数中加一个 data 选项，data 传一个对象进去，把对象转换成 & 和 = 连接的字符串，如果 type 是 post , 就把参数入在 send 方法中，并使用 xhr.setRequestHeader 设置请求头，如果 type是 get 就放在 url 后面

  3. 传递json参数： 在参数中加一个 header 选项，选项中加上 content-type, 如果 为 application/json 就在内部

    使用 JSON.stringy() 来处理 传递，再放在 send 方法中

  4. 处理错误：当状态码为 200 才调用成功的函数 ，否则才调用失败的 error 函数

  5. 处理响应：使用 xhr.getResponseHeader(‘Content-Type’) 方法，取到响应的类型，如果包含 application/json , 就使用 JSON.parse(xhr.responseText) 进行转换，再调用 success或 error 函数

  6. 优化参数：在函数内部 准备一个默认值，使用 Object.assign() 方法，把两个对象 合并

相关知识：

  1. 请回顾 实参 和 形参的知识
    1. 实参 调用函数时传递的参数， arguments
    2. 形参 定义函数时传递的参数 paramers
  2. 具体代码如下：
function ajax(options) {
    var xhr = new XMLHttpRequest();

    // 设置默认值
    var defaults = {
      type:'get',
      data: {}, 
      url:'',
      header: {
        'Content-Type' : 'application/json'
      },
      success: function() {},
      error: function() {}
    };

    // 用输入的值 覆盖 默认值
    Object.assign(defaults, options)

    // 先做字符 串拼接
    var params = '';

    for(var i in options.data) {
      params += (i +　'=' + options.data[i] + '&')
    }

    // 去掉最后一个 & parans.substr(0, params.length -1)
    params = params.slice(0, -1);

    if(options.type === 'get') {
      xhr.open(options.type, options.url + '?' + params);
    } else {
      xhr.open(options.type, options.url);

    }

    if(options.type === 'post') {
      var contentType = options.header['Content-Type']
      xhr.setRequestHeader('Content-type', contentType);
      if(contentType === 'application/json') {
        xhr.send(JSON.stringify(options.data))
      } else {
        xhr.send(params);
      }
    } else {
      xhr.send();
    }

    xhr.onload = function() {

      var contentType = xhr.getResponseHeader('Content-Type');


      var responseText = xhr.responseText;

      if(contentType.includes('application/json')) {

        responseText = JSON.parse(responseText);
      }
      // 进一步判断 状态码
      if(xhr.status === 200) {
        options.success(responseText);

      } else {
        options.error(responseText);

      }
    };
  }
